## 1. Design Philosophy

### 1.1 Core Principles (Updated for LLM Shell)

| Principle | Description |
|-----------|-------------|
| **Local-First Privacy** | 100% user data ownership. No data leaves device unless user-initiated export. No servers required. |
| **LLM-Agnostic** | Works with any LLM (ChatGPT, Gemini, Claude, Copilot). No vendor lock-in. |
| **Minimal Friction** | Copy-paste workflow should feel natural, not cumbersome |
| **Progressive Disclosure** | Complex prompts hidden behind simple "Generate" buttons |
| **Intelligent Simplicity** | App handles prompt engineering; users just learn |
| **Clear Feedback** | Every action has visible response. Errors explain what to do. |
| **Accessibility** | WCAG AA compliant, supports text scaling, screen readers |

## 7. Design Rationale

### 7.1 Why Copy-Paste Workflow?

The MVP uses a copy-paste workflow for LLM integration because:

1. **No API costs for users** - Users use their existing LLM subscriptions
2. **Works with any LLM** - ChatGPT, Gemini, Claude, Copilot, etc.
3. **Privacy** - No API keys stored, no server communication
4. **Simplicity** - Easy to implement, easy to understand
5. **Future-proof** - Can add direct API integration later

### 7.2 Why Local-First?

1. **True data ownership** - Users control 100% of their data
2. **No subscription required** - No recurring costs for app
3. **Works offline** - Study anywhere, anytime
4. **Privacy** - No user data on servers
5. **Reliability** - No server downtime affects learning

### 7.3 Why These Components?

| Component | Rationale |
|-----------|-----------|
| Flashcards | Proven SRS methodology for vocabulary |
| Quizzes | Active recall strengthens memory |
| Conversations | Contextual learning for real usage |
| Grammar | Structured rules need explanations |
