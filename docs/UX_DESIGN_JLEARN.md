## 1. Design Philosophy

### 1.1 Core Principles

| Principle | Description |
|-----------|-------------|
| **Offline-First** | Every interaction should work without network. No spinners waiting for content. |
| **Progressive Disclosure** | Show what's needed, reveal complexity gradually |
| **Motivation-Driven** | Celebrate progress, maintain streaks, reward consistency |
| **Japanese Aesthetic** | Clean, minimal, with subtle Japanese design elements |
| **Accessibility** | WCAG AA compliant, supports text scaling, screen readers |

## 5. Platform-Specific Considerations

### 5.1 Flutter (Android) Specifics

**Navigation:**
- Bottom navigation bar with 5 primary destinations
- Material 3 design system
- Android back gesture support
- Deep linking for notifications

**Gestures:**
- Swipe cards left/right for quick actions
- Pull-to-refresh on lists
- Long press for context menus
- Pinch to zoom on reading content

**Components:**
- Material 3 components (FilledButton, OutlinedButton)
- Adaptive layouts (phone/tablet)
- Custom animated flashcards
- Bottom sheets for options

**Offline:**
- All content cached in SQLite
- Audio files stored locally
- No network indicators on content screens
- Sync indicator only when connected

**Performance:**
- Lazy loading for lists (>100 items)
- Cached images with placeholders
- Preload audio for next items
- Background progress sync

- Responsive sidebar for desktop

- Browser back/forward support
